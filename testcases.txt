=========================================
TEST CASES: Laravel Web Application System
=========================================

[1] การลงทะเบียนผู้ใช้ (User Registration)
-----------------------------------------
Test Case ID: TC-REG-01
Description: ตรวจสอบการลงทะเบียนสำเร็จเมื่อกรอกข้อมูลครบถ้วนและถูกต้อง
Steps:
 1. เข้าไปที่หน้า /register
 2. กรอกชื่อ อีเมล รหัสผ่าน และ ยืนยันรหัสผ่าน (มีอย่างน้อย 8 ตัวอักษร)
 3. กดปุ่ม "Register"
Expected Result: ระบบบันทึกข้อมูล นำเข้าสู่ระบบอัตโนมัติ และ Redirect ไปยังหน้า Dashboard

[2] การเข้าสู่ระบบ (User Login)
-----------------------------------------
Test Case ID: TC-LOG-01
Description: ตรวจสอบการเข้าสู่ระบบสำเร็จ
Steps:
 1. เข้าไปที่หน้า /login
 2. กรอกอีเมลและรหัสผ่านที่ถูกต้อง
 3. กดปุ่ม "Log in"
Expected Result: ระบบเข้าสู่ระบบสำเร็จและเปลี่ยนหน้าเป็น Dashboard

[3] การรีเซ็ตรหัสผ่าน (Password Reset)
-----------------------------------------
Test Case ID: TC-PWD-01
Description: ตรวจสอบการขอลิงก์รีเซ็ตรหัสผ่านผ่านอีเมล (Forgot Password)
Steps:
 1. ที่หน้า /login กดที่ลิงก์ "Forgot your password?"
 2. กรอกอีเมลที่ใช้สมัครไว้แล้วกดส่ง
Expected Result: ระบบแจ้งเตือนว่าส่งลิงก์ไปยังอีเมลเรียบร้อยแล้ว ("We have emailed your password reset link.")

[4] การรักษาความปลอดภัย (Security & Permissions)
-----------------------------------------
Test Case ID: TC-SEC-01
Description: ตรวจสอบการเข้าถึงหน้าที่จำกัดสิทธิ์ในฐานะ Guest
Steps:
 1. ยังไม่ล็อกอิน ให้พิมพ์ URL /dashboard ตรงๆ
Expected Result: ระบบไม่อนุญาตให้เข้าถึง และ Redirect กลับมาหน้า /login อัตโนมัติ

[5] การแสดงผล Dashboard (รายรับ-รายจ่าย)
-----------------------------------------
Test Case ID: TC-DB-01
Description: ตรวจสอบการแสดงผลกราฟและตัวเลขยอดรวม
Steps:
 1. เข้าสู่ระบบและไปที่หน้า Dashboard
Expected Result: ต้องมีข้อมูลการสรุปผล "ยอดรายรับรวม" และ "ยอดรายจ่ายรวม" ปรากฏ และมีกราฟแท่ง (Bar Chart) เด้งขึ้นมา

Test Case ID: TC-DB-02
Description: ตรวจสอบการทำงานของตัวกรองช่วงเวลา (Date Filter)
Steps:
 1. ในหน้า Dashboard ระบุวันที่เริ่มต้น (Start Date) และวันที่สิ้นสุด (End Date)
Expected Result: กราฟและตัวเลขยอดรวมจะต้องอัปเดตแบบเรียลไทม์เพื่อแสดงเฉพาะข้อมูลในช่วงเวลาที่กำหนด (อ้างอิงจากข้อมูลที่อัปเดตล่าสุด)

Test Case ID: TC-DB-03
Description: ตรวจสอบสิทธิ์การเข้าถึงข้อมูลของ Super User
Steps:
 1. เข้าสู่ระบบด้วยบัญชีที่อยู่ในกลุ่ม Super User (is_super_user = 1)
Expected Result: มีเมนูตัวเลือก (Dropdown) "ผู้ใช้งาน" ปรากฏขึ้น สามารถเลือกดูข้อมูลรายรับรายจ่ายของ User อื่นๆ แบบเจาะจงได้

Test Case ID: TC-DB-04
Description: ตรวจสอบสิทธิ์การเข้าถึงข้อมูลของผู้ใช้ทั่วไป
Steps:
 1. เข้าสู่ระบบด้วยบัญชีผู้ใช้ธรรมดา
Expected Result: ไม่ปรากฏ Dropdown เพื่อเลือกชื่อบุคคลอื่น และข้อมูลที่แสดงผลเป็นเฉพาะข้อมูลส่วนตัวของผู้ใช้เองเท่านั้น

[6] ระบบสลับธีม (Dark / Light Mode)
-----------------------------------------
Test Case ID: TC-THM-01
Description: ตรวจสอบฟังก์ชันการเปลี่ยนธีมและการจดจำสถานะ (Local Storage)
Steps:
 1. เข้าหน้าเว็บ คลิกปุ่มเปลี่ยนธีม (รูปพระอาทิตย์/พระจันทร์) มุมบนขวา
 2. สังเกตสีพื้นหลัง สีข้อความ ในหน้าเว็บ กราฟ และ Sidebar ว่าสอดคล้องกัน
 3. กด Refresh (F5) หน้าจอ หรือปิดเว็บบราวเซอร์แล้วเปิดใหม่
Expected Result: ธีมจะสลับมาอีกโหมดอย่างสมบูรณ์ และเมื่อ Refresh แล้วเว็บจะยังคงจดจำการตั้งค่าธีมก่อนหน้าเอาไว้

[7] การรองรับอุปกรณ์เคลื่อนที่ (Responsive UI & Mobile)
-----------------------------------------
Test Case ID: TC-RES-01
Description: ตรวจสอบการปรับหน้าจอสำหรับมือถือและแท็บเล็ต
Steps:
 1. ย่อหน้าต่าง Web Browser ลงมาให้มีความกว้างเทียบเท่าโทรศัพท์มือถือ
Expected Result: แถบ Sidebar จะหายไปอัตโนมัติ องค์ประกอบต่างๆ เช่น ช่องตัวกรอง (Filters) กราฟ และตาราง จะปรับขนาดไม่ล้นหน้าจอ (ไม่พัง)

Test Case ID: TC-RES-02
Description: ตรวจสอบการใช้งาน Hamburger Menu
Steps:
 1. ในสภาวะหน้าจอมือถือ (ย่อหน้าต่าง) ให้กดไอคอน 3 ขีด (Hamburger Menu) มุมขวาบน
 2. สังเกตรายชื่อลิงก์
Expected Result: เมนูย่อยลิงก์ไปหน้าต่างๆ (Dashboard, Ledgers, Profile) ต้องปรากฏขึ้น และถ้าเป็น Super User ต้องมีลิงก์ Group Users และ Users ให้กดใช้งานด้วย
